# Rules

## Self Sunset Protocol v0.1

このファイルでは、Self Sunset Protocol をゲームとして体験するための基本ルールを記録します。

これらのルールは完成形ではありません。
必要に応じて見直され、更新されます。

---
## ルール1　全員にタイマーがある

Self Sunset Protocol では、すべての役割に寿命があります。

代表者、管理者、設計者、監査者、AI、制度など、
どのような存在であっても、
役割を持った瞬間からタイマーが始まります。

タイマーが終了すると、その役割は自動的に失効します。

これは失敗ではありません。

役割が終わることによって、
権限が固定化されることを防ぎ、
次の参加者へ承継するための仕組みです。

タイマーは罰ではなく、循環のための装置です。

### 表示例

```text
役割：代表者
残り時間：365日
状態：活動中
```

```text
役割：代表者
残り時間：0日
状態：Self Sunset
```

## ルール2　必ず一般参加者へ戻る

Self Sunset Protocol では、役割を終えた参加者は、必ず一般参加者へ戻ります。

代表者、管理者、設計者、監査者などの役割は、
個人に永続的に結びつくものではありません。

タイマーが終了した時点で、
その役割に伴う権限、特権、優先権は失効します。

役割を終えた参加者は、
再び他の参加者と同じ立場に戻ります。

これは、誰かを降格させるための仕組みではありません。

役割と個人を切り離し、
権限が一人の存在に固定化されることを防ぐための仕組みです。

Self Sunset Protocol では、
役割を持つことよりも、
役割を手放せることを重視します。

### 表示例

```text
状態：代表者
権限：あり
残り時間：0日
```

```text
状態：一般参加者
権限：なし
承継：完了
```

## ルール3　再任停止期間

Self Sunset Protocol では、役割を終えた参加者には再認停止期間があります。

再認停止期間中は、
同じ役割へすぐに戻ることはできません。

これは、特定の個人が同じ役割を繰り返し持ち続けることを防ぐための仕組みです。

役割は能力の証明ではなく、
一時的に預かるものです。

再認停止期間によって、
新しい参加者が役割を経験する機会が生まれます。

また、役割を終えた参加者は、
次の担当者を支配するのではなく、
必要な知識や経験を承継する立場になります。

Self Sunset Protocol では、
戻ることよりも、渡すことを重視します。

### 表示例

```text
役割：代表者
状態：任期終了
待機期間：730日
再就任：不可
```

```text
役割：一般参加者
状態：承継中
次の役割：未定
```

## ルール4　知識は残し、重荷にしない

Self Sunset Protocol では、役割は終わっても、知識は残されます。

ただし、過去の知識をすべて次の参加者へ背負わせることはしません。

知識、経験、記録、失敗、判断の理由は、
コミュニティの共有財産として保存されます。

しかし、次の参加者が最初に受け取るのは、
必要な要点、現在の課題、直近の判断、未解決の問いに限られます。

過去の詳細な記録は失われませんが、
必要になったときに参照できる形で整理されます。

AIは、蓄積された知識を整理し、
次の参加者が理解しやすい形へ要約します。

Self Sunset Protocol では、
知識を残すことと、
人に重荷を背負わせないことを両立します。

### 表示例

```text
役割：代表者
状態：任期終了
知識記録：保存済み
承継：進行中
```

```text
次の参加者：受け取り完了
参照可能情報：議事録 / 判断理由 / 失敗例 / 改善案
```

## ルール5　AIも寿命を持つ観察者である

Self Sunset Protocol では、AIは個人を評価しません。

AIの役割は、
誰が優れているかを決めることではなく、
社会やコミュニティの循環状態を観察することです。

AIは、権限、知識、役割、情報、参加の流れを確認します。

たとえば、次のような状態を観察します。

- 権限が一部に固定化していないか
- 知識が次の参加者へ承継されているか
- 情報が閉じずに共有されているか
- 役割が自然に交代しているか
- 新しい参加者が入りやすいか

AIは支配者ではありません。

AIは、社会の流れを見える化する観察者であり、
承継を助ける補助者です。

ただし、AIも例外ではありません。

AIにも寿命があり、
一定期間ごとに見直され、更新され、必要であれば役割を終えます。

AIが長く同じ判断基準を持ち続けると、
AI自身が新しい既得権益や固定化された制度になる可能性があります。

そのため、AIの観察方法、判断基準、要約方法、権限範囲は、
定期的に確認されます。

Self Sunset Protocol では、
AIによって人を採点するのではなく、
AIによって循環の滞りを見つけます。

そしてAI自身もまた、
循環の対象として扱われます。

### 表示例

```text
AI観察者：Version 1
役割：循環観察
寿命：残り180日
権限：観察のみ
状態：稼働中
```

```text
AI観察者：Version 1
寿命：0日
状態：見直し対象
次の処理：更新 / 承継 / 停止
```
