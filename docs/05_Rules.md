# Rules

## Self Sunset Protocol v0.1

このファイルでは、Self Sunset Protocol をゲームとして体験するための基本ルールを記録します。

これらのルールは完成形ではありません。
必要に応じて見直され、更新されます。

---
## ルール1　全員にタイマーがある

Self Sunset Protocol では、すべての役割に寿命があります。

代表者、管理者、設計者、監査者、AI、制度など、
どのような存在であっても、
役割を持った瞬間からタイマーが始まります。

タイマーが終了すると、その役割は自動的に失効します。

これは失敗ではありません。

役割が終わることによって、
権限が固定化されることを防ぎ、
次の参加者へ承継するための仕組みです。

タイマーは罰ではなく、循環のための装置です。

### 表示例

```text
役割：代表者
残り時間：365日
状態：活動中
```

```text
役割：代表者
残り時間：0日
状態：Self Sunset
```

## ルール2　必ず一般参加者へ戻る

Self Sunset Protocol では、役割を終えた参加者は、必ず一般参加者へ戻ります。

代表者、管理者、設計者、監査者などの役割は、
個人に永続的に結びつくものではありません。

タイマーが終了した時点で、
その役割に伴う権限、特権、優先権は失効します。

役割を終えた参加者は、
再び他の参加者と同じ立場に戻ります。

これは、誰かを降格させるための仕組みではありません。

役割と個人を切り離し、
権限が一人の存在に固定化されることを防ぐための仕組みです。

Self Sunset Protocol では、
役割を持つことよりも、
役割を手放せることを重視します。

### 表示例

```text
状態：代表者
権限：あり
残り時間：0日
```

```text
状態：一般参加者
権限：なし
承継：完了
```

## ルール3　再任停止期間

Self Sunset Protocol では、役割を終えた参加者には再認停止期間があります。

再認停止期間中は、
同じ役割へすぐに戻ることはできません。

これは、特定の個人が同じ役割を繰り返し持ち続けることを防ぐための仕組みです。

役割は能力の証明ではなく、
一時的に預かるものです。

再認停止期間によって、
新しい参加者が役割を経験する機会が生まれます。

また、役割を終えた参加者は、
次の担当者を支配するのではなく、
必要な知識や経験を承継する立場になります。

Self Sunset Protocol では、
戻ることよりも、渡すことを重視します。

### 表示例

```text
役割：代表者
状態：任期終了
待機期間：730日
再就任：不可
```

```text
役割：一般参加者
状態：承継中
次の役割：未定
```

## ルール4　知識は残し、重荷にしない

Self Sunset Protocol では、役割は終わっても、知識は残されます。

ただし、過去の知識をすべて次の参加者へ背負わせることはしません。

知識、経験、記録、失敗、判断の理由は、
コミュニティの共有財産として保存されます。

しかし、次の参加者が最初に受け取るのは、
必要な要点、現在の課題、直近の判断、未解決の問いに限られます。

過去の詳細な記録は失われませんが、
必要になったときに参照できる形で整理されます。

AIは、蓄積された知識を整理し、
次の参加者が理解しやすい形へ要約します。

Self Sunset Protocol では、
知識を残すことと、
人に重荷を背負わせないことを両立します。

### 表示例

```text
役割：代表者
状態：任期終了
知識記録：保存済み
承継：進行中
```

```text
次の参加者：受け取り完了
参照可能情報：議事録 / 判断理由 / 失敗例 / 改善案
```

## ルール5　AIも寿命を持つ観察者である

Self Sunset Protocol では、AIは個人を評価しません。

AIの役割は、
誰が優れているかを決めることではなく、
社会やコミュニティの循環状態を観察することです。

AIは、権限、知識、役割、情報、参加の流れを確認します。

たとえば、次のような状態を観察します。

- 権限が一部に固定化していないか
- 知識が次の参加者へ承継されているか
- 情報が閉じずに共有されているか
- 役割が自然に交代しているか
- 新しい参加者が入りやすいか

AIは支配者ではありません。

AIは、社会の流れを見える化する観察者であり、
承継を助ける補助者です。

ただし、AIも例外ではありません。

AIにも寿命があり、
一定期間ごとに見直され、更新され、必要であれば役割を終えます。

AIが長く同じ判断基準を持ち続けると、
AI自身が新しい既得権益や固定化された制度になる可能性があります。

そのため、AIの観察方法、判断基準、要約方法、権限範囲は、
定期的に確認されます。

Self Sunset Protocol では、
AIによって人を採点するのではなく、
AIによって循環の滞りを見つけます。

そしてAI自身もまた、
循環の対象として扱われます。

### 表示例

```text
AI観察者：Version 1
役割：循環観察
寿命：残り180日
権限：観察のみ
状態：稼働中
```

```text
AI観察者：Version 1
寿命：0日
状態：見直し対象
次の処理：更新 / 承継 / 停止
```

## ルール6　次の担い手は循環から浮上する

Self Sunset Protocol では、
次の担い手を毎回の投票、推薦、抽選、立候補だけで決めることを前提にしません。

役割、知識、情報、参加の流れを観察し、
その循環の中から、次の担い手となり得る参加者が浮上します。

次の担い手は、
単に人気がある人や声の大きい人ではなく、
その役割に必要な知識や関係性の線上に自然に現れている参加者です。

AIは、循環の流れを観察し、
次の担い手の候補を見える化します。

人は、その候補が適切かどうかを確認し、
必要に応じて修正します。

この仕組みは、
選ぶことの負担を減らし、
役割の承継をより自然な流れとして扱うためのものです。

Self Sunset Protocol では、
次の担い手は上から任命されるのではなく、
循環の中から浮かび上がるものとして扱います。

## 表示例

```text
役割：地域調整者
状態：任期終了間近

循環上の候補：
- 参加者A：知識接続 82%
- 参加者B：関係接続 74%
- 参加者C：承継準備 68%
```
```text
AIの役割：
候補の浮上と理由の説明

人の役割：
確認、修正、異議申し立て
```

### ルール7　既得権益指数

Self Sunset Protocol では、
社会やコミュニティの循環が滞っていないかを観察するために、
既得権益指数を用います。

既得権益指数は、
特定の個人や集団を責めるための数値ではありません。

それは、
権限、知識、役割、情報、参加の流れが、
どれだけ固定化しているかを見える化するための指標です。

たとえば、次のような状態が続くと、
既得権益指数は上昇します。

- 同じ参加者が同じ役割を何度も担っている
- 知識が一部の参加者にしか共有されていない
- 情報が閉じた場所で扱われている
- 次の担い手が浮上しにくい
- 新しい参加者が入りにくい
- AIの判断基準が長期間見直されていない
- 制度そのものが更新されずに固定化している

既得権益指数が高くなった場合、
Self Sunset Protocol は、
役割、制度、AI、知識の流れを見直します。

この指数は、
人を裁くためではなく、
循環の詰まりを見つけるために使われます。

Self Sunset Protocol では、
既得権益を個人の悪として扱うのではなく、
社会の流れが硬くなった状態として観察します。


### 表示例

```text
既得権益指数：12%
状態：健全

循環はおおむね開かれています。
```

```text
既得権益指数：48%
状態：注意

一部の役割と知識が固定化し始めています。
承継と担い手の浮上を見直してください。
```

```text
既得権益指数：82%
状態：停滞

権限、知識、制度の固定化が強くなっています。
Self Sunset の発動条件を確認してください。
```

### AI統制指数

既得権益指数とあわせて、AI統制指数を観察します。

AI統制指数とは、
AIが社会やコミュニティの判断、整理、承継、選出、評価に
どれだけ強く関与しているかを見える化する指標です。

AI統制指数が高くなるほど、
人間の判断や参加の余地が少なくなり、
AI自身が新しい固定化された制度になる可能性があります。

たとえば、次のような状態が続くと、
AI統制指数は上昇します。

- AIが候補者を提示するだけでなく、実質的に決定している
- AIの判断基準が公開されていない
- AIの要約だけが読まれ、元の記録が参照されない
- AIの提案に異議を申し立てる仕組みがない
- AIの寿命や更新時期が見直されていない
- 人間がAIなしでは意思決定できなくなっている

AI統制指数は、
AIを否定するための指標ではありません。

AIが支配者にならず、
観察者、司書、補助者として機能しているかを確認するための指標です。

Self Sunset Protocol では、
AIもまた循環の対象であり、
AIの影響力も見える化されます。

### 表示例

```text
AI統制指数：18%
状態：健全

AIは観察と要約を中心に機能しています。
人間による確認と修正が維持されています。
```

```text
AI統制指数：56%
状態：注意

AIの提案が意思決定に強く影響しています。
判断基準の確認と、人間側の異議申し立て経路を見直してください。
```

```text
AI統制指数：84%
状態：危険

AIが実質的な決定者になっています。
AIの寿命、権限範囲、判断基準を見直してください。
```


## 達成条件　循環が続いていること

Self Sunset Protocol における達成条件は、
特定の個人が勝利することではありません。

達成とは、
参加者、役割、知識、制度、AIが入れ替わりながらも、
社会やコミュニティの循環が続いている状態を指します。

プレイヤー自身が役割を終え、
権限や立場を手放したあとも、
知識が承継され、
次の担い手が浮上し、
制度が見直され続けているなら、
その社会は達成状態に近づいていると考えます。

Self Sunset Protocol では、
個人の勝利ではなく、
循環の継続を達成として扱います。

### 循環率の段階

```text
0% - 20%
状態：停止

権限、知識、制度が固定化し、
新しい参加や承継がほとんど起きていない状態。

21% - 40%
状態：停滞

一部の循環はあるものの、
特定の参加者、制度、AIに依存している状態。

41% - 60%
状態：維持

最低限の承継や役割交代は行われているが、
循環にはまだ偏りや詰まりがある状態。

61% - 80%
状態：良好

役割、知識、制度、AIが適度に更新され、
個人に依存しすぎずに機能している状態。

81% - 100%
状態：自律循環

プレイヤー自身が役割を終えたあとも、
社会やコミュニティが自律的に承継と更新を続けられる状態。
```

```markdown
### 注意

循環率が100%であることを常に目指す必要はありません。

重要なのは、
社会が止まらず、
人や制度やAIが固定化されず、
必要に応じて見直され続けることです。
```

### 達成の目安

循環率が一定期間 61%以上を維持し、
既得権益指数とAI統制指数が高くなりすぎていない場合、
その社会は達成状態に近いと判断します。

最終的な達成は、
プレイヤー自身がいなくなったあとも、
循環率が維持され続けることです。


## 例
循環率：74%
状態：良好

既得権益指数：18%
AI統制指数：22%
承継状態：安定

## 制度にも寿命を持たせる

Self Sunset Protocol では、
人や役割だけでなく、制度にも寿命があります。

制度は、一度作られたら永遠に残るものではありません。

その制度が作られた理由、
必要とされた時代、
支えていた参加者、
守ろうとしていた価値は、
時間とともに変化します。

そのため、制度には有効期間を設定します。

有効期間が終了した制度は、
自動的に見直しの対象となります。

見直しでは、次のいずれかを選びます。

- 継続する
- 修正する
- 統合する
- 停止する
- Archiveへ移す

制度を終了することは失敗ではありません。

役目を終えた制度を静かに手放すことで、
新しい制度や新しい参加者が生まれる余地ができます。

Self Sunset Protocol では、
制度を守り続けることではなく、
制度が必要な間だけ機能し、
必要がなくなれば次へ渡ることを重視します。

### 表示例

```text
制度：地域調整制度
寿命：残り365日
状態：有効
見直し予定：2027-07-07
```

```text
制度：地域調整制度
寿命：0日
状態：見直し対象

選択肢：
- 継続
- 修正
- 統合
- 停止
- Archiveへ移す
```

制度は、永遠に残るためではなく、必要な時間だけ社会を支えるために存在します。


## AIにも寿命を持たせる

Self Sunset Protocol では、
制度と同じように、AIにも寿命があります。

AIは一度導入されたら永遠に使われる存在ではありません。

AIの観察方法、判断基準、要約方法、権限範囲は、
時間とともに固定化し、
新しい既得権益や見えない支配を生む可能性があります。

そのため、AIには有効期間を設定します。

有効期間が終了したAIは、
自動的に見直しの対象となります。

見直しでは、次のいずれかを選びます。

- 継続する
- 更新する
- 分割する
- 権限を縮小する
- 停止する
- Archiveへ移す

AIを終了することは失敗ではありません。

AIが役目を終えることによって、
人間の判断、別のAI、新しい観察方法へと承継する余地が生まれます。

Self Sunset Protocol では、
AIを永続的な支配者として扱うのではなく、
一定期間だけ循環を支える観察者・補助者として扱います。

AIもまた、
循環の対象であり、
Self Sunset の例外ではありません。

### 表示例

```text
AI観察者：Version 2
寿命：残り180日
状態：稼働中
権限：観察 / 要約 / 候補整理

AI観察者：Version 2
寿命：0日
状態：見直し対象

選択肢：
- 継続
- 更新
- 分割
- 権限を縮小
- 停止
- Archiveへ移す
```

AIを永続的な支配者にしないことが、Self Sunset Protocol の基本原則です。


## 稲妻を循環で受け流す

Self Sunset Protocol では、
社会やコミュニティに突然の稲妻が起こることを前提とします。

経済危機、制度の失敗、外部環境の変化、技術の変化、
参加者の離脱、AIの誤作動などによって、
社会の流れに大きな衝撃が走る場合があります。

しかし、そのたびに強制的な介入を行うことは、
新しい権力の乱用につながる可能性があります。

そのため、Self Sunset Protocol では、
稲妻を単純に押さえ込むのではなく、
循環の中で受け流し、分散し、発散させることを重視します。

稲妻が発生した場合、
まず次の状態を確認します。

- 権限が一部に集中していないか
- 知識が一部に閉じていないか
- 制度が硬直していないか
- AIの判断が強くなりすぎていないか
- 次の担い手が浮上できる状態か
- 役割や制度の寿命が見直されているか

Self Sunset Protocol では、
強制的に誰かを排除することを第一の手段とはしません。

代わりに、
役割の分散、知識の承継、制度の見直し、
AIの権限縮小、担い手の浮上によって、
衝撃を社会全体へ分散させます。

稲妻のような変化は起こります。

しかし、その稲妻を一つの場所に落とすのではなく、
循環の網の中で受け流し、
新しい流れへ変えていくことを目指します。


## 負の循環を止めるための装置

Self Sunset Protocol では、
加害行為によって負の循環が生まれた場合、
それを放置しません。

負の循環を止め、
修復循環へ戻すために、
次の装置を用います。

### 1. 被害の即時保護

加害行為が確認された場合、
最初に行うのは判断や処罰ではなく、
被害者の安全を確保することです。

必要に応じて、
接触制限、距離の確保、情報遮断、生活支援、心理的支援を行います。

### 2. 負の循環指数

負の循環がどれだけ広がっているかを観察するために、
負の循環指数を用います。

負の循環指数は、
被害、報復、孤立、沈黙、再加害、情報の歪みが
どれだけ連鎖しているかを示す指標です。

この指数は、
誰かを責めるためではなく、
連鎖を止める場所を見つけるために使います。

### 3. 責任の可視化

加害者には、
何が起きたのか、
誰にどのような影響を与えたのか、
どの責任を負うのかを明確にします。

責任の可視化は、
加害者を永久に烙印化するためではなく、
再発防止と修復の出発点として扱います。

### 4. 修復条件

被害者は、
安全、説明、謝罪、距離、補償、再発防止、
記録の保存などについて、
必要な条件を提示できます。

ただし、
修復条件は復讐の制度化ではありません。

被害者、加害者、第三者、AIのいずれか一つに
最終決定権を集中させず、
記録と対話を通じて調整します。

### 5. 再発防止と再参加の条件

加害者が再び社会やコミュニティに参加する場合、
単に時間が経過したことだけではなく、
責任の理解、行動の変化、教育、距離の尊重、
被害者の安全が確認される必要があります。

再参加は自動的に認められるものではなく、
修復循環の中で慎重に扱われる状態です。


## 生存基盤の循環指標

Self Sunset Protocol では、
食料や住居のような生存基盤について、
循環状態を指標として表示します。

### 表示例

```text
食料自給循環率：90%
状態：安定

地域内で必要な食料の多くが生産・共有・再分配されています。


廃棄食料循環率：90%
状態：良好

廃棄される食料の多くが、
再利用、再分配、堆肥化、資源化されています。


住居循環率：68%
状態：改善中

空き家、公共住宅、共有住宅の一部が、
必要な参加者へ再配分されています。
```

ここで大事なのは、**所有を完全否定しないこと**です。  
所有を全否定すると、自由や生活の安定も壊れやすいです。SSPでは、

```text
所有してよい。
ただし、固定化・独占・投機化したら観察対象になる。

使ってよい。
使わなくなったら手放せる。

公共的に選べる住居の層を作る。
でも、すべてを強制的に奪わない。
```
